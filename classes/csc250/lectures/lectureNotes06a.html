NFA computation (tree)

We can visualize what happens with the computation of a string with an NFA by building a NFA computation Tree.

This is like "unrolling an FA as we walk it into paths", except with two extra rules:

  1. You add a possible path into a "next" state for every outgoing symbol option, and
  2. You add a possible path into a "next" state any states we might get to through an arrow labeled with \(\epsilon\)

DFA-NFA equivalence

Activity 7 [2 minutes]:

Are NFAs stronger than DFAs?






Let's make a claim!

NFAs and DFAs are equally powerful!





Let's proove it by construction:

  • What we know: \(M_{NFA}=(Q,\Sigma,\delta, q_0, F)\)
  • Need to construct: \(M_{DFA}=(Q^\prime,\Sigma,\delta^\prime, q_0^\prime, F^\prime)\) that recognizes L(M)


Let's start with the states \(Q \rightarrow Q^\prime \)

It‚Äôs possible that in an NFA, we could be in multiple states at each step. If we want to emulate the NFA‚Äôs behavior, we‚Äôll have to have a state for each combination.





Now the Start states \(q_0 \rightarrow q_0^\prime\)





(In this case, only \(A\))



Now the Accepting states \(F \rightarrow F^\prime\)





(In this case, \( \{ C, AC, BC, ABC \} \))



Now the Transitions \(\delta \rightarrow \delta^\prime\)

At each symbol reading, if there are multiple paths to follow, we want to jump the the single state (in the DFA) that is the Union of all the possible destination states (in the NFA) for that symbol.

This can be expressed like this: \[ \begin{alignat}{2} &\delta^\prime ( q, \mathit{a} ) = \bigcup\limits_{p\in q} \delta(\mathit{p}, a) \\ &\qquad \text{ where $q \in Q^\prime$; $a \in \Sigma$} \\ &\qquad \text{ and $q$ can be a set of states from $Q$} \end{alignat} \]
