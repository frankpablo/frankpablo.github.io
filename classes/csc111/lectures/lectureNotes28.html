Lecture Notes 27: More about Recursion




Advanced Recursion: Memorization

In many cases, recursive programs go over work that was done before.

One technique is to store results for work that was performed before and simply look it up when available.

Why do we need this?

Check out this program:

def fibonacci(input_value):
  if input_value == 1:
    return 1
  elif input_value == 2:
    return 1
  elif input_value > 2:
    return fibonacci(input_value -1) + fibonacci(input_value -2)

# fast
for i in range(1, 11):
   print("fib({}) = ".format(i), fibonacci(i))

######## Fibonacci (no memo)

fibonacci_cache = {}

def fibonacci_memo(input_value):
  if input_value in fibonacci_cache:
    return fibonacci_cache[input_value]
  if input_value == 1:
      value = 1
  elif input_value == 2:
      value = 1
  elif input_value > 2:      
      value = fibonacci_memo(input_value -1) + fibonacci_memo(input_value -2)
  fibonacci_cache[input_value] = value
  return value

for i in range(1, 201):
   print("fib({}) = ".format(i), fibonacci_memo(i))

Moving the turtle's starting position

It is annoying that the image starts on the center, so we can move the starting position of the turtle like this: 

import turtle

window = turtle.Screen()
window.setup(400, 400)
t = turtle.Turtle()

#### start: moves starting position
t.penup()
t.goto(-window.window_width()/4, -window.window_height()/4)
t.pendown()
#### end: moves starting position

t.fd(50)
t.right(90)
t.fd(50)
t.right(90)
t.fd(50)
t.right(90)
t.fd(50)
t.right(90)
