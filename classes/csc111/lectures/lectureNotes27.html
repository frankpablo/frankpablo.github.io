Lecture Notes 27: Fancy graphics with user-objects (Lab)




Designing and building a complex multi-class project

Today, we'll design, develop, and test, a slightly more complex graphics-based project.

Steps

  • Before we start... sit with your final project teams
  • For each task, one should code while the others supervise / direct the steps
  • The person coding should not be the person making the "what to code next" decisions


    • The Task:


    1. We will choose a type of scene to draw that contains:
      • multiple identical complex objects
      • placed randomly in a scene
    2. We will design the classes that need to be created to complete this project
    3. We will develop in modules:
      • We will develop base classes first --- and TEST them (using ad-hoc testing functions)
      • We will develop any inherited classes next --- and TEST them (using ad-hoc testing functions)
      • We will develop the implementation code that uses these classes in small steps --- while TESTING them (using ad-hoc testing functions)
