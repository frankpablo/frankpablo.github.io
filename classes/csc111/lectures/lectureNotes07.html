Recap: Simple Functions and the Execution Flow

First: we will finish the activities from the previous lecture if we did not finish then.

Then: Recap of the execution flow with a simple function: 

# function definition
  def next(num):
    return num + 1

  # statement area
  nxt = next(2)
  print(nxt)
  print("done!")
Use the Python Tutor

Things to notice:
  1. the def block is saved but not executed
  2. execution follows starts in a top-bottom sequence (for now!)
  3. the execution jumps to the body of the def block when an invocation is made, changing the control flow
  4. the "parameter" num "receives" a value, it is used, and then... what happens to it?
  5. after the function is executed, we return where we left off (after the invocation


Why Functions?

As we said:

Scope

Scope refers to the "area of influence" of a variable or name.
It is the block of code in which it is "alive" and where it could be used.

If you notice the code for our example, num exists for a brief period of time.

It's "alive" from the moment the function next is invoked and the argument (\(2\)) is inserted into the function parameter num.

We say that the variable num is bound to 2.

Lastly, num is no longer saved after the function returns.

The Global Scope

As you can see from where some variables are visualized in the Python Tutor, there is something called The Global Scope!

The global scope corresponds to the maximum scope of the python program (the most external one).

creating variabls at this level let's any other part of the program interact with it.

Global variables are frowned upon because of this.

Some reasons for this is that one might have unexpected side effects from functions using a global variable.

if you see the global keyword used inside a function, that is telling Python to look for the variable in the global scope.
This opens up the chance for that function to modify the variable.

ZyBooks mentions and uses them, but we'll try to avoid using them as we learn more about functions and scope.

The general rule is: A variable's scope is that of the innermost function, class or module in which they're assigned.

We will see more details about scope when we see Conditionals (next class).



Arguments, Parameters, and Local Variables

As you saw above, we have several values being saved into variables and use differently depending on their scope.

First, a couple definitions:

Arguments


In the code shown above, 
x = addThree(x)

x is present both as an argument (value or variable passed to a function) of the function addThree,
and as the Left-Hand-Side variable in the assignment.

The order of execution is:
  1. First resolve the value of x
  2. then copy that value into the parameter used inside the function
  3. Once the function returns, assign the return value to the Left-Hand-Side variable of the assignment.


Parameters


In the function definition for addThree, we have:

Which has a single parameter that will be received as input.

Once a function has been invoked with arguments, the values of the arguments are copied into the appropriate parameters and used inside the function as local variables (with the scope local to the function only).

You can also declaree additional variables inside the function and use them while inside the function.

The main Function

There is a way to force Python to start execution inside a function.

This is useful for two reasons:

  1. First, that let's us have a "starting scope" that is NOT the Global scope
  2. It let's you use the file in another program without executing it!!!.
    Example: imagine you want to quote a phrase from a famous song (let's say "The Galaxy Song", by Eric Idle) but you don't remember it.
    In this case, you want to look it up (say in google), and use it.
    You don't want for it to actually start playing!


So how do we get the program to start inside a function?

Forcing the start of a main function

The following construction is standard in a Pyhton program that we want to be able to

2) import into another python program (and use any functions defined in it)

# Does caesar cipher encoding for a single letter
def getCipher ( my_letter, my_int):
    print ('inside getCipher') # this is not necessary
    upper_letter = my_letter.upper() 
    ascii_num = ord(upper_letter) 
    alphabet_num = ascii_num - 65 # A's ascii num is 65
    new_alphabet_num = alphabet_num + my_int # <-- Q1
    new_alphabet_num = new_alphabet_num % 26 # <-- Q2
    new_ascii_num = new_alphabet_num + 65
    new_upper_letter = chr(new_ascii_num)
    return new_upper_letter

    # Action section

    let = getLetter ('Gimme a letter: ') 
    cipher = getCipher( let, x ) # Note the order of the arguments
    print('The plaintext is: ', let)
    print('The ciphertext is: ', cipher)

The Branch!

The concept of a branch is simple but of great importance.

A Branch is a jump in the flow of execution into a block of code that is one among a set of options, including "doing nothing".

For Example: Imagine you're sitting in your couch, listening to music, and you get a call on the phone.

Here are your options:

  1. Do nothing
  2. Turn the music off
  3. Answer the phone


You could decide to do one or many of these in some sequence, like:
  • Do nothing (for a while); Then Answer the phone
  • Turn the music off
  • Turn the music offl Then Answer the phone
  • Do nothing


There are different ways to phrase the distinct options, like What if I want to do only one?, or What if I am allowed to skip some but do the others?

As you may imagine, in terms of the execution of the program, a brach is changing the execution flow depending on the outcome of the brach conditions.

We've already seen we can follow "non-linear" execution flows (with function calls); now, we'll simply add conditions to establish different possible execution flows.
