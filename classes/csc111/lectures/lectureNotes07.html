Lecture Notes 07: More about Functions




Recap: Simple Functions and the Execution Flow

First: we will finish the activities from the previous lecture if we did not finish then.

Then: Recap of the execution flow with a simple function: 

# function definition
def next(num):
  return num + 1

# statement area
nxt = next(2)
print(nxt)
print("done!")
Use the Python Tutor

Things to notice:
  1. the def block is saved but not executed
  2. execution follows starts in a top-bottom sequence (for now!)
  3. the execution jumps to the body of the def block when an invocation is made, changing the control flow
  4. the "parameter" num "receives" a value, it is used, and then... what happens to it?
  5. after the function is executed, we return where we left off (after the invocation


Why Functions?

As we said:
  1. Reuse: Avoid unnecessary repetition
  2. Reduce: Modular structure, better organization
  3. Recycle: Use functions that you created a while ago for something you need now!



Scope

Scope refers to the "area of influence" of a variable or name.
It is the block of code in which it is "alive" and where it could be used.

If you notice the code for our example, num exists for a brief period of time.

It's "alive" from the moment the function next is invoked and the argument (\(2\)) is inserted into the function parameter num.

We say that the variable num is bound to 2.

Lastly, num is no longer saved after the function returns.
