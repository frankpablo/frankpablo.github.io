Clarifying Example: company Census

Some of the confusion about Recursion lies in the fact that it usually feels like it operates "in reverse".

This is because a subrouting is in "pause" until it finishes with the recursive calls it made.



Another issue is that "all recursive calls" start a new subroutine that performs a new function call (with its own stack frame).



The following example shows how this works:



Company Census Recursive Call (with humans)

The following is an org-chart for "Recursive Industries":



  1. If a "group" has no branches under them, they simply report their number to the requesting group
  2. If they do,
    1. it's going to ask each of its branches to report their total numbers.
    2. Once they do, it will add them up (plus their own number) and report the sum to the requesting group

Moving the turtle's starting position

It is annoying that the image starts on the center, so we can move the starting position of the turtle like this: 

import turtle

window = turtle.Screen()
window.setup(400, 400)
t = turtle.Turtle()

#### start: moves starting position
t.penup()
t.goto(-window.window_width()/4, -window.window_height()/4)
t.pendown()
#### end: moves starting position

t.fd(50)
t.right(90)
t.fd(50)
t.right(90)
t.fd(50)
t.right(90)
t.fd(50)
t.right(90)
